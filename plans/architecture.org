* DiffyDuck: Side-by-Side Terminal Diff Viewer

** Project Goals

- Side-by-side terminal diff viewer with treesitter syntax highlighting
- Fold/unfold capability for chunks, files, or other units
- Future: parse treesitter nodes to identify affected classes/functions
- Uses git commands directly (not a pager) to enable custom features

** Testing Philosophy

TDD everything. The previous attempt failed because interactive parts weren't testable.

*** Testing Layers

| Layer | What                        | How                              |
|-------+-----------------------------+----------------------------------|
|     1 | Pure business logic         | Regular Go unit tests            |
|     2 | Update logic (state changes)| Unit tests on Update function    |
|     3 | View rendering              | Golden file snapshots            |
|     4 | Interactive flows           | teatest / catwalk                |

*** Key Principle

Test behavior through *model state*, not rendered output.
- Scroll test: assert ~model.scrollOffset == 5~, not parse screen
- Fold test: assert ~model.hunks[0].folded == true~, not look for "[+]"
- Use golden tests only for visual regression, not logic verification

** Phase 1: Minimal Side-by-Side Viewer [COMPLETE]

*** Goal

Run ~git show <commit>~ and display a side-by-side view of changes.
No syntax highlighting. No folding. Just the core viewing loop.

*Status: All items in this phase are complete.*

*** What We Need

1. *Diff Parser* (~pkg/diff~)
   - Parse unified diff format from git show output
   - Extract: file names, hunks, added/removed/context lines
   - Map line numbers for left (old) and right (new) sides

2. *Git Interface* (~pkg/git~)
   - Run ~git show~ and capture output
   - Interface-based for test mocking

3. *Side-by-Side Model* (~pkg/sidebyside~)
   - Transform parsed diff into paired lines (old, new)
   - Handle: additions (blank left), deletions (blank right), modifications, context

4. *TUI Layer* (~internal/tui~)
   - Model: holds side-by-side lines, scroll position, viewport size
   - Update: handle scroll keys (j/k/up/down/pgup/pgdn), quit (q)
   - View: render two columns with line numbers
