* Help Screen (C-h) — DONE

** Goal
Press C-h anywhere (except text-editing modes) to open a full-screen help pager
that replaces the diff view. Press q / Esc / C-g / C-h to dismiss and return.
The help pager scrolls with the same movement keys as the main diff (j/k, f/b,
d/u, pgup/pgdn, G/gg, etc.).

** Decisions
- C-h active in: normal mode, visual mode, help mode (toggles off)
- C-h NOT active in: comment mode (C-h = backspace), search input mode
- Binding data: single central registry via =KeyMap.BindingGroups()=. All groups
  (including g-sequences, C-w sequences, search input, comment editing, visual
  mode) defined together in keys.go.
- g-sequences folded into relevant sections (gg → Navigation, gj/gk → Navigation)
- Search input bindings merged into the Search group
- Mode-entry keys duplicated: V in both Actions and Visual Mode, Enter in both
  Actions and Comment Editing
- Opposite bindings paired on one line with "/" separator (e.g. "Scroll down / up")
- "/" separator styled with fg=15
- Description shown first, keys second (e.g. "Scroll down / up   ↓  j / ↑  k")
- Two-column layout when terminal is wide enough; column 2 starts based on
  actual max width of column 1 content, with 4-char gap
- Layout rebuilds on terminal resize
- Fixed bottom bar: "q, Esc, C-g or C-h to go back"
- Title: "diffyduck help" (lowercase)

** Data structures (keys.go)

#+begin_src go
type Binding struct {
    Keys  []string // display keys, e.g. ["j", "↓"] or ["gg"]
    Desc  string   // human-readable description
    Keys2 []string // optional: paired opposite keys
    Desc2 string   // optional: paired opposite description
}

type BindingGroup struct {
    Name     string
    Bindings []Binding
}
#+end_src

=KeyMap.BindingGroups()= returns all 7 groups: Navigation, Search, Folds & Files,
Actions, Window Management, Visual Mode, Comment Editing.

=AllBindingGroups(km)= is a convenience wrapper.

** Model changes (model.go)
- =helpMode bool= — when true, View() and Update() switch to help behaviour
- =helpScroll int= — scroll offset within help content
- =helpLines []string= — pre-rendered plain help lines (for scroll counting)

** Key handling (update.go)
Order in =handleKeyMsg()=:
1. Pending-key checks (g, ctrl+w) — must come first so gg works in help mode
2. Comment mode guard — C-h is backspace here
3. Search mode guard — C-h not intercepted
4. C-h toggle — enters/exits help mode
5. Help mode dispatch → =handleHelpKey()=
6. Visual mode / normal mode

** Help rendering (help.go)
- =handleHelpKey()= — scroll (j/k/f/b/d/u/G/gg/pgup/pgdn), exit (q/Esc/C-g/C-c)
- =buildHelpGroupBlocks()= — renders each group with aligned desc→key layout,
  paired bindings with styled "/" separator
- =splitBlocksIntoColumns()= — balances groups into two columns by total line count
- =leftColumnMaxWidth()= — measures actual left column content width
- =canFitTwoColumns()= — checks leftWidth + gap + 20 ≤ termWidth
- =mergePlainColumns()= / =mergeStyledColumns()= — side-by-side column merging
- =renderHelp()= — title bar + scrolled content + fixed bottom bar
- =formatKeyForDisplay()= — up→↑, ctrl+x→C-x, etc.
- Resize: =WindowSizeMsg= rebuilds =helpLines= and clamps scroll
