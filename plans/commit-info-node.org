#+TITLE: Commit Info as Foldable Child Node
#+DATE: 2025-01-25

* Summary

Transform the commit body content (sha, author, date, message) into a foldable
child node that appears as the first child under each commit, using yellow
shaders (like files but yellow-colored).

* Requirements

- New child node under each commit, appearing FIRST (before files)
- Header line: "commit" (lowercase) with yellow ~▒~ shaders
- Two fold states: ~◯~ (header only) and ~●~ (full content)
- Follows parent commit fold level:
  - ~CommitFolded~ → node hidden
  - ~CommitNormal~ → header only
  - ~CommitExpanded~ → full content (commit sha, Author, Date, subject, body)
- Tab on info header toggles between Normal and Expanded
- gj/gk navigation should work with the commit info node
- First file border visibility depends on commit info expanded state

* Implementation (completed)

** Row types (~view.go~)
- Added ~RowKindCommitInfoHeader~ and ~RowKindCommitInfoBody~
- Added ~isCommitInfoHeader~ and ~isCommitInfoBody~ flags to ~displayRow~
- Added ~commitInfoLine~ field for body content

** Row building (~view.go~)
- Created ~buildCommitInfoRows()~ function
  - Returns empty if ~CommitFolded~
  - Returns only header row if ~CommitNormal~
  - Returns header + body rows if ~CommitExpanded~
- Header is just "commit" (lowercase)
- Body contains: blank line, "commit <full sha>", Author, Date, blank, subject, body

** Rendering (~view.go~)
- Created ~renderCommitInfoHeader()~ - yellow ~▒~ shaders, no bold text
- Created ~renderCommitInfoBody()~ - 2-space gutter (dedented from original 4)

** Fold toggle (~update.go~)
- Added ~isOnCommitInfoHeader()~ function
- Added ~handleCommitInfoFoldToggle()~ function
- Updated ~handleFoldToggle()~ to check for commit info header first
- Fixed ~setAllCommitsToLevel()~ to use ~CommitExpanded~ for level 3

** Navigation (~update.go~)
- Updated ~goToNextHeading()~ to include commit info header as navigable node
- Updated ~goToPrevHeading()~ to include commit info header as navigable node
- gk from first file now goes to commit info header (not commit header)

** Border visibility (~view.go~)
- First file's ~prevFileUnfolded~ now checks if commit info is expanded
- Diff view compatibility: defaults to true when no commit metadata
- Fixed file fold toggle to only upgrade commit to ~CommitNormal~ (not ~CommitExpanded~)

** Cursor identity (~update.go~)
- Added ~commitInfoBodyIndex~ to ~cursorRowIdentity~
- Added tracking for commit info body rows in ~getCursorRowIdentity()~
