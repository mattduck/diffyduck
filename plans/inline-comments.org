#+TITLE: Inline Comments Feature
#+DATE: 2026-01-23

* MVP [DONE]

** Scope
- [X] Comments on add/delete/changed lines on the NEW side only (left column)
- [X] One comment per line
- [X] Comments are ephemeral (lost on exit)
- [X] Comment editing happens at bottom of screen (multi-line prompt)
- [X] Box border around inline comments

** UX Flow
1. [X] User presses Enter on a commentable line
2. [X] Status bar area becomes a multi-line text input prompt
3. [X] User types comment (multi-line supported via Enter for newlines)
4. [X] Submit: C-j
5. [X] Cancel: C-c or C-g
6. [X] Edit existing: Press Enter on line with comment to edit it
7. [X] Delete: Remove all text and submit

** Keybindings in Comment Mode
- [X] Normal typing: Insert characters
- [X] Enter: Insert newline
- [X] C-j: Submit comment
- [X] C-a: Beginning of line
- [X] C-e: End of line
- [X] C-f / Right: Forward char
- [X] C-b / Left: Backward char
- [X] C-k: Kill to end of line
- [X] C-c / C-g: Cancel comment
- [X] Backspace: Delete backward
- [X] Delete: Delete forward
- [X] Space: Insert space

** Display
- [X] Comment appears inline in the diff, under the commented line
- [X] Spans only the new/left side (right side shows shading)
- [X] Has a visible box border (┌─┐ │ │ └─┘)
- [X] Bright white text and borders (fg=15)
- [X] When file is folded, comments are hidden (no indicator for MVP)
